# MyAppDemo

字节工程训练营客户端方向作业

## 核心功能
点击start开始

LoginPageDemo： 实现注册及登录功能，通过Room存储用户信息，登录后进入个人页面

FeedPageDemo： 展示已注册的用户信息，支持下拉刷新，滚动到底时加载更多，长按删卡功能

## 版本更新
### v1.0.0   架构调整(2025/11/30 )
  - **更新了feed流部分的整体架构**，保证单一职责:  
    adapter负责管理UI，拆分出DataManager负责管理数据，拆分出ViewHolder，支持后续插件式添加卡片样式（还没做注册方法）
  - 添加了下拉加载时的loading动画
  
### v1.1.0   优化个人信息页面(2025/11/30 )
  - 优化了修改昵称，现在点界面内其他地方也会结束编辑
  - 添加了头像，可以点击“生成头像”生成随机头像，头像由服务端下发
  - 添加了两个开关，控制是否在Feed流界面展示自己的头像和视频（具体卡片样式还没实现，视频之后计划使用倒计时模拟）
  - 添加了编辑视频时长功能，模拟视频
  
### v1.2.0   实现卡片注册，添加新样式(2025/12/1 )
  - 在FeedPage中添加了单双列切换开关，可选择单列或双列排版
  - **添加了卡片注册功能**，支持插件式扩展，需要新建卡片类和ViewHolder类，然后在activity中注册即可。
  - 加入了一个新的包含头像的卡片布局，可以与原布局混排。

### v1.3.0   实现卡片曝光事件回调(2025/12/2 )
  - 为FeedPage中的卡片加入了曝光事件回调，包含以下事件：开始露出、露出50％、完整露出、消失，可以在滚动时和布局发生改变时进行检测
  - 在FeedPage中加入了一个滚动的文本框，直接显示每张卡片的曝光时间

### v1.4.0   添加视频卡片(2025/12/6 )
  - 加入了一个包含视频的卡片布局，视频通过一个倒计时模拟，包含进度条和重播按钮
  - 该卡片始终保持单列（可与其他双列卡片混排）
  - 视频卡片完整露出时自动播放
### v1.4.1   优化视频卡片(2025/12/6 )
- 修改了卡片消失事件的逻辑，现在曝光时间回调只需要遍历可见的卡片了，提高了性能
- 现在视频卡片消失时会停止播放
- 修复了加载动画不出现的bug
### v1.5.0   架构调整(2025/12/7 )
- 重点优化了FeedActivity，解耦出了两个可复用的工具类，FeedListManager是一个RecycleView管理器，CardExposureManager是一个曝光事件管理器，部分加载数据的逻辑放进了ViewModel里，同时也对activity中的代码进行了整理，看起来更清楚
### v1.6.0   添加缓存功能(2025/12/7 )
- 添加了缓存功能，进入MainActivity时，对图片数据进行缓存，断网时使用本地缓存展示
- 修复了个人页面两个开关不记录默认状态的bug
### v1.6.1   性能优化(2025/12/7 )
